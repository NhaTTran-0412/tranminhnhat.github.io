Git and GitHub

  • Git

    • Git is distributed version control system designed to handle all projects of different scales. It has functions such as source code archiving, restoring source code of any version, helping to coordinate project implementation in a team effectively.
    Git has many benefits to attracting users, such as being easy to learn, and great support for teamwork. Furthermore, it also makes managing and synthesizing the developers' code easier.

  • GitHub

    • GitHub is a web hosting platform for projects that use the Git control system. It is mainly used for projects where many users collaborate and monitor project changes, and GitHub can also restore code as needed.
    GitHub provides many social networking functions for developers to learn from each other's work experience

      Basic command: config, add, commit, push

  • Git config

    • git config is used to set up your username and email inmain configuartion file
    command:
    git config --global user.name " Your GitHub username"
    git config --global user.email " Your GitHub email"

  • Git add

    • Function: Add file content to the index
    command:
    Git add .

  • Git commit

    • Function: Record changes to the repository
    command:
    Git commit -m "Message"

  • push

    • Function: Update remote references along with associated objects
    command:
    Git push

      How to resolve conflict

    conflict in Git is generally a conflict that occurs when two files are merged with the same content. Git will mark the file as a conflict and pause the merging, it is the developer's responsibility to resolve the conflict.
    To resolve the conflict, you need to edit the conflicted file so that it no longer conflicts with the previous file. after the file has been edited, use the git add command to create a new merge and git commit. Git will see that conflict is resolved

      Branching

    Branch is a function which is used to branch and record the history. branch will not affect another branch, so you can perform many changes in the same repository.
    Some basic command with branch
    Create branch : Git branch newbranch
    Checkout branch : Git checkout branch
    Delete branch : Git -d branch
